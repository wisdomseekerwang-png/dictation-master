## 听写大师 - 修复 TXT 导入错误 + Redis 数据损坏问题

- 问题：iPad 导入 TXT 文件后，词库显示 {"error_code":31142 processing please wait} 这类词语
- 根本原因：Upstash Redis 免费版有 QPS 限制，限流时返回错误 JSON，被当作正常词库数据保存
- 修复1：api/data.ts 的 Redis 客户端增加响应校验，error/error_code 字段时返回 null
- 修复2：App.tsx 的 loadFromServer 加入 sanitizeWordBank，过滤掉 JSON 字符串格式的"词语"
- 修复3：store.ts 的 parseWords 改为按行解析，支持"词语 中文释义"格式（英文词+释义只取第一个词）
- 用户需要在 iPad 上先删除那个损坏的词库 PET0413,_1，然后重新导入
- 跨设备同步 bug：另一台电脑看不到词库
  - 原因：loadFromServer 的合并逻辑有缺陷，当服务器返回空词库时错误地回退到本地空数据
  - 修复：改为判断"服务器是否有实际数据"（词库非空 或 错词非空 或 记录非空），有数据时直接用服务器数据
  - TXT 文件 PET0413,_2.txt 格式：中文繁体词语，用中文标点（，。、！）或空格分隔，每行可能有多个词语
  - 修复后正确解析：`肯定地 毫无疑问地明确地确定地` → ["肯定地", "毫无疑问地", "明确地", "确定地"]
  - 逻辑：把所有中文标点+英文逗号+空格替换为换行，再按行提取词语
  - 再次优化 parseWords：支持两种模式
  - 格式A（优先）：编号句子跨多行（如 unit6.txt），先合并换行为空格，再用正则按"数字."分割出完整句子
  - 格式B：普通词语/句子，按标点和空格拆分
  - unit6.txt 解析示例：8个完整句子，每句40-80词
    - 例1：`今天天气真好，我们去公园吧。` → ["今天天气真好", "我们去公园吧"]
    - 例2：`肯定地 毫无疑问地明确地确定地,` → ["肯定地", "毫无疑问地", "明确地", "确定地"]
    - 例3：`hello world,你好世界！` → ["hello world", "你好世界"]
